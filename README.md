<h1 align="center">VEIL Windows GUI</h1>

<p align="center">A quiet Windows client for the VEIL private tunnel: GUI window, tray, service, and MSI installer.</p>

---

`veil-windows-gui` is the user-facing Windows client. It is intentionally small:
one connection state, the active endpoint, traffic counters, last handshake,
config selection, and `veil://` import. The data plane runs in a Windows service
so the tunnel can start at boot and survive the GUI being closed.

## Components

- `veil-gui.exe` - Fyne desktop window + tray icon. Double-click opens the app.
- `veil-service.exe` - Windows service that owns the TUN adapter, routing, DNS,
  UDP socket, and VEIL engine.
- `veiltun.dll` - Wintun-compatible driver DLL loaded by the service through the
  VEIL-owned `veiltun` binding from [veil-windows](https://github.com/veil-proto/veil-windows).
- `VEIL-*.msi` - x64 installer that installs the GUI, service, and driver DLL.

## Install

Download the latest MSI from
[Releases](https://github.com/veil-proto/veil-windows-gui/releases) and run it.
The installer registers `VEILTunnel` as a Windows service and starts it.

After install, open **VEIL** from the Start menu or by double-clicking
`veil-gui.exe`.

## Import A Config

The GUI accepts full `veil://` config links generated by
[veil-install](https://github.com/veil-proto/veil-install):

1. Copy a `veil://...#Name` link.
2. Open VEIL.
3. Paste it into the import field and press **Import link**.
4. Select the imported config and press **Connect**.

Imported configs are stored under `%APPDATA%\VEIL\configs` as plain `.conf`
files. The service persists the active config under `%ProgramData%\VEIL` so the
tunnel reconnects after reboot.

## Portable Build

For development:

```powershell
go build -ldflags "-H windowsgui" -o veil-gui.exe ./cmd/veil-gui
go build -o veil-service.exe ./cmd/veil-service
copy driver\amd64\veiltun.dll .
.\veil-service.exe install   # elevated shell
.\veil-gui.exe
```
